Case overview

Task

The system parameters cannot be managed in a chronological order. In addition, Playbook are not componentized, thus automation/efficiency of the work in construction and operation not realized.

Introduction of IT Automation

Solution

Using the function of Exastro IT Automation

  • Configuration management of system parameters
  • Component management of Ansible Playbook

By connecting the phases of design, construction, and operation, automation of work is realized.

Effects after introduction

Effects

Example 1: Used in monitoring setup operation
Before configuration management / automation

Workload of 4 billion yen is carried out per year.

After configuration management / automation

Can now be implemented in a workload of 1 billion yen per year. (75% reduction)

Example 2: Used in command list maintenance operation
Before configuration management / automation

Workload of 40 million yen is carried out per year.

After configuration management / automation

Can now be implemented in a workload of 8 million working load per year. (80% reduction)
The convenience was also improved by the immediate reflection because the lead time until the maintenance is no longer required.

Case overview

Task

Initial configuration is required to perform on a large amount of network devices. Howerver, manual operation via serial console requires much workload, also there is limit to manage large amount of configuration value by using spreadsheet software.

Introduction of IT Automation

Solution

By aggregating serial console with SmartCS, configuring multiple devices simultaneously via SSH becomes possible. Furthermore, by calling Ansible and central managing a large amount of configuration value with Exastro IT Automation, automating the initial configuration of network devices also becomes possible.

Effects after introduction

Effects

Automated initial configuration of 6 types of network devices such as switch, etc., achieving release of 2,000 devices per month.

Case overview

Task

In large-scale carrier systems, analyzing the impact to service due to planned or sudden accident device shut-down required much workload.

Introduction of IT Automation

Solution

By managing the system configuration using the function of Exastro IT Automation, automatically predicting the service impact due to device shut-down becomes possible.

*Impact prediction is custom developed by using the screen framework provided by ITA.

Effects after introduction

Effects

About 800 thousand yen of workload/cost is required for performing anaysis one time, the workload are not required anymore by this method. Since analysis is conducted about 120 times per year, the effect of reducing about 94 million yen / year is confirmed.
